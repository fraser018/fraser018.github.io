HTML, CSS, Javascript and the DOM

An analogy to describe the differences between HTML and CSS

HTML and CSS are both vitally important when designing a website today. A straight forward way to think about the two is that HTML is your a web pages main structure. Whereas CSS is the way that the page ends up looking, with information regarding colours and theming.

Whilst HTML handles the core structure of the page, CSS is able to adjust this structure quite dramatically. With the growing disparity between screen sizes and resolutions across all over our devices, CSS can help to resize HTML elements so that they display correctly no matter what screen you are using.!

Control Flow and Loops

In programming the control flow of code is literally the controlling of where and what our code does.JavaScript is executed line by line from top to bottom, unless something in our code specifically changes that flow.

A simple example of control flow would be using loops, we can essentially ask our code if a situation is true before continuing to the next part. If this is not true then we can tell the code to loop back around until something changes.

A real life example of this could be like the basic checks we do on a daily basis. For example checking if there is milk in the fridge before making coffee; if there is then we continue to make a coffee but if there isn't then we would go and get some from the shop first before continuing to make the coffee.

The DOM (Document Object Model)

The DOM (Document Object Model) is a way of navigating and manipulating elements of HTML on a live website.These changes are primarily made using JavaScript code.

Modifying the code on a website using the DOM allows us to create more interactive websites for the users. The DOM can be used to modify basically anything on the site.An example of this could be that you have an 'Accept' button for a form but you want this button to be disabled by default. Only after the user has ticked a box would the button activate and allow the user to click it.

Arrays vs Objects

We use objects and arrays to store data and then access this data at another time. The two are very similar to each other but generally speaking an array would be used for ordered collections of data. Being able to access the data based on it's numerical location in the array. An object is usually used to represent a real world 'object' where it's characteristics can be broken down.



Functions

Functions are a huge part of JavaScript, without them JS would not work.

They are basically small collections of code that can be run and return an answer. Functions can be run over and over again so that they do not need to be re-written in the code. This way a function can be specified at the start of the code and later re-called by the code to get the answer we need.
